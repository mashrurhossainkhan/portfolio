LeetCode #3 Sliding Window HashMap / Map

Longest Substring Without Repeating Characters

A step-by-step blog explanation of the JavaScript solution using sliding window and a Map.

1. Problem in simple words

We are given a string s. We need to find the length of the longest substring that contains no repeated characters.

Important: substring means characters must stay continuous. For example, in "pwwkew", "wke" is a substring, but "pwke" is not, because it skips characters.

2. Example

s = "abcabcbb"

Answer = 3

Because the longest valid substring can be "abc".

3. Brute force idea

Try every possible substring, check whether it has duplicates, and keep the maximum length.

This works, but it is slow.

For interviews, brute force is usually not the best final answer here.

4. Best idea: Sliding Window

We keep a window of characters that is always valid, meaning no duplicates inside it.

  • left = starting index of the current window
  • right = ending index of the current window
  • We move right one step at a time
  • If a duplicate appears, we move left forward
Think of it like a moving rectangle over the string. The rectangle expands to the right, and when it becomes invalid, we shrink it from the left.

5. Why do we need a Map?

The Map stores the last index where each character was seen. 

lastSeen.set(char, index)

Example:

For "abc":
a -> 0
b -> 1
c -> 2

This helps us jump left directly to the correct position when we see a duplicate.

6. Final code

/**
 * @param {string} s
 * @return {number}
 */
var lengthOfLongestSubstring = function (s) {
  const lastSeen = new Map();
  let left = 0;
  let maxLength = 0;

  for (let right = 0; right < s.length; right++) {
    const char = s[right];

    if (lastSeen.has(char) && lastSeen.get(char) >= left) {
      left = lastSeen.get(char) + 1;
    }

    lastSeen.set(char, right);
    maxLength = Math.max(maxLength, right - left + 1);
  }

  return maxLength;
};

7. Step-by-step explanation of every line

const lastSeen = new Map();

This stores each character and the last position where we saw it.

let left = 0;

This is the start of the current window.

let maxLength = 0;

This stores the best answer found so far.

for (let right = 0; right < s.length; right++)

We move through the string one character at a time using right.

const char = s[right];

Current character at the right pointer.

if (lastSeen.has(char) && lastSeen.get(char) >= left)

This checks whether the current character was already seen inside the current window.

The >= left part is very important. It makes sure we only care about duplicates that are still inside the current window.

left = lastSeen.get(char) + 1;

If duplicate exists in the current window, move the left pointer to one position after that duplicate.

lastSeen.set(char, right);

Update the latest index of the current character.

maxLength = Math.max(maxLength, right - left + 1);

The current window length is right - left + 1. Compare it with the best answer.

return maxLength;

Return the final answer.

8. Dry run with "pwwkew"

Step right char left after update Window maxLength Reason
1 0 p 0 "p" 1 First char, no duplicate
2 1 w 0 "pw" 2 No duplicate
3 2 w 2 "w" 2 Duplicate 'w' found, move left
4 3 k 2 "wk" 2 No duplicate
5 4 e 2 "wke" 3 Best window so far
6 5 w 3 "kew" 3 Duplicate 'w' again, move left
Final answer: 3, because the longest substring without repeating characters is "wke" or "kew".

11. Why lastSeen.get(char) >= left matters

Suppose the string is "abba".

When we are at the last 'a', the old 'a' might be outside the current window. We should only move left if that repeated character is still inside the current window.

Without >= left, we might move left backward by mistake, which breaks the logic.

12. Time and space complexity

Complexity Value Why
Time O(n) Each character is processed in one pass
Space O(k) Map stores characters seen in the string

13. Interview explanation you can say

I use a sliding window and a hashmap.
The window keeps a substring with no repeated characters.
I move the right pointer through the string.
If I see a duplicate character inside the current window,
I move the left pointer to one index after the previous occurrence.
The hashmap stores the last seen index of each character,
so I can update left in O(1) time.
At each step, I update the maximum window length.
This gives O(n) time complexity.

14. Final takeaway

The main trick is not removing characters one by one manually. Instead, use the last seen index to jump the left pointer directly.

This problem is one of the most important sliding window interview questions. If you understand this deeply, many similar problems become easier.
